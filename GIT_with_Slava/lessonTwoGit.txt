  1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

для этого применяем команду:      git checkout <имя файла>


2.Как вернуть файл из индекса в рабочий каталог?

для этого применяем команду:      git restore --staged <имя файла> или вместо имени .  тогда будут возвращены все файлы


3.Как отменить ошибочный commit ?

для этого применяем команду:      git reset HEAD~1 (где 1 это колличество коммитов)



4.Как создать новую ветку и переключится в нее?

для этого применяем команду:      git branch <имя ветки>
                                  git checkout -b <имя ветки> - это тоже самое только позволяет создать ветку и сразу в нее переключиться.
                                  git branch -v  -  позволяет посмотреть , какие ветки существуют.


5.В чем отличение между fast-forward merge  and non fast-forfard merge?

fast-forward merge - это когда после разветвлении веток работа ведется в одной из них, а затем происходит их слияние.

non fast-forfard merge - это когда после разветвлении веток работа ведется в обеих ветках, а затем происходит их слияние.


6.Когда может возникнуть конфликт слияния веток, как  его разрешить?

При использовании non fast-forfard merge может произойти конфликт, если не завершена работа в одной из веток (т.е. не закончен коммит) либо 
коммиты не совпадают. Разрешить проблему можно путем выбора приорететных данных из одной из веток через редактор (nano например и 
удаления не нужных).



7.Можно ли удалить ветку? Как?

Удалить ветку можно. для этого применяем комманду :      git branch -d <имя ветки>
